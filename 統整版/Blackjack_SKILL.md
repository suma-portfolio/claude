## 一、遊戲規則核心知識庫

### 1.1 標準 Blackjack 流程（State Machine）

```
[IDLE]
  │
  ▼
[BETTING]          ← 玩家下主注 + 可選邊注
  │
  ▼
[DEALING]          ← 玩家兩明牌；荷官一明一暗
  │
  ├─ 荷官明牌 = A ──▶ [INSURANCE]   ← 可買保險（主注½，賠率 2:1）
  │                        │
  │                    確認後繼續
  │
  ▼
[PLAYER_DECISION]  ← Hit / Stand / Double / Split / Cash Out
  │
  ├─ 爆牌 (>21) ──▶ [BUST]  → 直接結算
  ├─ 21 ──────────▶ [AUTO_STAND]
  ├─ Split ────────▶ [SPLIT_HAND_1] → [SPLIT_HAND_2]
  │
  ▼
[DEALER_REVEAL]    ← 翻開暗牌
  │
  ▼
[DEALER_DRAW]      ← 依桌規補牌
  │
  ▼
[SETTLEMENT]       ← 計算結果 + 派彩
  │
  ▼
[IDLE]
```

---

### 1.2 牌值計算

| 牌面 | 點數 |
|---|---|
| 2–10 | 面值 |
| J / Q / K | 10 |
| A | 1 或 11（取有利值）|

- **Soft Hand**：含 Ace 且可視為 11 而不爆牌（例：A+6 = Soft 17）
- **Hard Hand**：不含 Ace，或 Ace 只能算 1（例：A+6+K = Hard 17）

---

### 1.3 標準決策選項

| 動作 | 條件 | 說明 |
|---|---|---|
| **Hit** | 任何時候 | 要一張牌 |
| **Stand** | 任何時候 | 停牌 |
| **Double Down** | 初始兩張牌 | 加倍注碼，只補一張牌 |
| **Split** | 兩張同點數 | 分成兩手，各補一張；通常限一次 |
| **Surrender** | 初始兩張牌（部分桌規） | 放棄半注回收 |
| **Insurance** | 荷官明牌 A | 額外注（主注½），荷官 BJ 賠 2:1 |
| **Cash Out** | 特定變異版本 | 提前兌現（動態倍率）|

---

### 1.4 派彩表（標準）

| 情況 | 賠率 |
|---|---|
| Natural Blackjack（前兩張 = 21） | **3:2** |
| 一般勝 | 1:1 |
| 平局（Push） | 退注 |
| 荷官 Blackjack（無保險） | 輸主注 |
| 保險中獎 | 2:1 |
| 加倍後勝 | 1:1（實際贏 2× 主注）|

---

### 1.5 荷官規則

| 規則 | 說明 |
|---|---|
| Stand on Hard 17 | 大多數桌規標準 |
| Hit on Soft 17 (H17) | 對莊家更有利，提高莊家優勢約 0.2% |
| Peek（美式） | 荷官查看暗牌，若 BJ 立即結算 |
| No Peek（歐式） | 荷官不查看，玩家須冒完牌後才知道 |

## 二、變異玩法全覽

### 2.1 Infinite Blackjack（無限21點）

```
核心差異：
- 所有玩家共享同一組手牌（Four Card Final 前）
- 四張牌後可選 Four Card Final（1:1 結算）或繼續
- Cash Out 功能：倍率 0.4–1.77×
- Six Card Charlie：六張不爆自動勝
- 無牌靴限制（連續洗牌機 CSM）
```

**邊注（Side Bets）：**
| 邊注 | 說明 |
|---|---|
| Perfect Pairs | 前兩張牌相同點數 |
| 21+3 | 玩家兩張 + 荷官明牌組合（撲克牌型）|
| HOT3 | 玩家兩張 + 荷官明牌點數加總 |
| Bust Bonus | 荷官爆牌時依張數賠付 |

---

### 2.2 Spanish 21

```
- 移除所有 10 點牌（僅保留 J/Q/K）→ 對莊家有利
- 補償：玩家 21 永遠勝（即使荷官也 21）
- 特殊牌型獎勵（5張21、6張21、7-7-7等）
- 可 Double 後 Surrender（Late Surrender after Double）
- 可 Re-split Aces
```

---

### 2.3 Pontoon（英式 / 亞洲版）

```
- 兩張明牌對玩家，荷官兩張暗牌（玩家不知荷官手牌）
- Pontoon = Natural BJ，賠 2:1（最高牌型）
- Five Card Trick = 5張不爆，賠 2:1（次高）
- 買牌（Hit）= "Twist"，停牌（Stand）= "Stick"
- 加倍（Double）= "Buy"
- 平局荷官贏
```

---

### 2.4 Super Fun 21

```
- 玩家 21 永遠勝
- 鑽石 BJ（♦A + ♦J）賠 2:1
- 玩家可 Double 任何張數後，再 Surrender
- 允許 Re-split（含 Aces）
- Natural BJ 賠 1:1（除鑽石 BJ），賠率降低補償規則寬鬆
```

---

### 2.5 Free Bet Blackjack

```
- 特定手牌可「免費」Double 或 Split（莊家支付）
- 免費 Double：Hard 9、10、11
- 免費 Split：除 10 點牌以外任何對子
- 推注規則（Push 22）：荷官 22 點時，玩家非 BJ 一律平局
- 降低了策略容錯壓力，適合休閒玩家
```

---

### 2.6 Lightning Blackjack（Evolution）

```
- 每局隨機抽取「閃電牌」，對應 2×–25× 倍率乘數
- 閃電牌只在玩家手牌中生效
- 需繳交額外費用（Lightning Fee）才能享受乘數
- 增加娛樂性和刺激感，適合直播博弈情境
```

---

### 2.7 其他常見變異

| 名稱 | 核心差異 |
|---|---|
| Double Exposure | 荷官兩張明牌；平局荷官勝；BJ 賠 1:1 |
| Blackjack Switch | 可交換兩手牌的第二張；Push 22 規則 |
| Vegas Strip | 4 副牌、Peek、Re-split Aces |
| Multihand | 同時玩 2–5 手，各手獨立決策 |
| Progressive BJ | 連結累積彩池邊注 |

## 四、UI/UX 設計原則

### 4.1 遊戲 UX 核心法則

```
UI（User Interface）= 玩家看到什麼、能點什麼
UX（User Experience）= 玩家在整個流程中的感受與決策效率

博弈遊戲的 UX 目標：
  1. 降低認知負荷（不讓玩家迷失在哪個階段）
  2. 加速決策循環（減少操作步驟）
  3. 強化即時回饋（每個動作有明確的視覺/聽覺響應）
  4. 建立情緒張力（贏的爽感、輸的不甘心 → 繼續遊玩）
```

---

### 4.2 遊戲階段 × UI 狀態對應

| 遊戲階段 | 主要 UI 元件 | 可互動元素 | 鎖定元素 |
|---|---|---|---|
| IDLE | 空桌面、歡迎訊息 | 入座按鈕 | 所有遊戲控制 |
| BETTING | 籌碼面板、投注區、邊注區 | 籌碼選擇、投注區點擊、清除/重注 | 決策按鈕 |
| DEALING | 牌飛行動畫 | 無（等待） | 所有控制 |
| INSURANCE | 保險詢問、倒數計時 | Yes / No 按鈕 | 主決策按鈕 |
| PLAYER_DECISION | 決策按鈕組、手牌、計分 | Hit/Stand/Double/Split/Cash Out | 投注區、籌碼 |
| DEALER_REVEAL | 荷官翻牌動畫 | 無（等待） | 所有控制 |
| DEALER_DRAW | 荷官補牌動畫 | 無（等待） | 所有控制 |
| SETTLEMENT | 結果 Toast、派彩動畫 | 重注/新局按鈕 | 決策按鈕 |

---

### 4.3 決策按鈕 UX 設計規範

#### 按鈕狀態
```
Default   → 可用，清晰呈現
Hover     → 視覺放大或高亮（Desktop）
Active    → 按下瞬間縮放反饋
Disabled  → 灰化 + 降低透明度（不可用時，如 Double 注碼不足）
Selected  → 已選中狀態（保險 Yes/No）
```

#### 按鈕排列邏輯（由左至右優先順序）
```
[Split]  [Double]  [Hit]  [Stand]

說明：
- Hit / Stand 最常用 → 放置最顯眼位置（右側）
- Double / Split 次之 → 條件滿足才亮起
- Cash Out / Surrender → 特殊條件才出現，不佔常態位置
```

#### 禁用條件
| 按鈕 | 禁用條件 |
|---|---|
| Double | 餘額不足 / 已 Hit 過 / Split Aces 後 |
| Split | 兩張牌不同點數 / 已 Split 過 / 餘額不足 |
| Cash Out | Aces Split 後 / 未達條件 |
| Insurance YES | 餘額 < 主注½ |

---

### 4.4 籌碼面板 UX

```
設計原則：
1. 面額排列由小到大，最常用面額放中央
2. 選中籌碼有明確高亮效果
3. 點擊投注區 = 疊加選中籌碼
4. 右鍵 / 長按 = 移除一枚籌碼（降低誤操作損失）
5. 清除鈕 = 一鍵清空（需確認 or 快速動作）
6. 重注鈕 = 帶入上局注碼（節省操作）
```

---

### 4.5 計分顯示 UX

```
顯示規則：
- Soft Hand：顯示兩個可能值，例「7/17」
- Hard Hand：顯示單一數值
- 爆牌：紅色數值 + BUST 字樣
- Blackjack：金色高亮 + BJ 標記
- Six Card Charlie：特殊標記

位置：
- 永遠跟隨手牌組顯示（避免玩家視線移動）
- 荷官分數：暗牌未翻時只顯示明牌點數，翻牌後動態更新
```

---

### 4.6 回饋動畫規範

| 事件 | 動畫類型 | 時長 | 說明 |
|---|---|---|---|
| 發牌 | 滑入 + 旋轉 | 300–500ms/張 | 模擬真實發牌感 |
| 翻牌 | 水平翻轉 | 400ms | 3D flip |
| 爆牌 | 晃動 + 紅色閃爍 | 600ms | 強調負面結果 |
| Blackjack | 金光閃爍 + 放大 | 800ms | 強化勝利感 |
| 贏局 | 籌碼飛入 + 數字滾動 | 1–2s | 派彩動畫 |
| 輸局 | 籌碼飛出（暗化） | 600ms | 視覺上「帶走」注碼 |
| 平局 | 灰色 Toast | 400ms | 低調處理 |
| Cash Out | 鎖定效果 + 倍率展示 | 500ms | 確認感 |

---

### 4.7 計時器 UX（倒數設計）
倒計時從 15 秒開始

```
二段式顏色：
  > 6秒  → 綠色（安全）
  < 5秒  → 紅色閃爍（緊迫）

超時行為：
  - BETTING 超時 → 自動下重注 or 不下注（依桌規）
  - INSURANCE 超時 → 自動選 No（視為放棄保險）
  - PLAYER_DECISION 超時 → 自動 Stand（保護玩家）

計時器位置：
  - 靠近當前需互動的元件（不讓玩家視線游移）
  - Mobile：置頂或置於按鈕上方
```

---

### 4.8 Split 多手牌版型

```
排列原則：
  - 兩手並列顯示，左右對稱
  - 當前決策手有明確高亮邊框
  - 完成的手牌降低亮度（表示「已處理」）
  - 各手牌獨立顯示計分

限制提示：
  - Split Aces 後：各補一張 → 自動 Stand → 不可 Cash Out
  - 視覺上鎖定額外操作按鈕 + Tooltip 說明原因
```

## 五、常見 UX 陷阱 × 解決方案

| 問題 | 說明 | 解決方案 |
|---|---|---|
| 階段不明 | 玩家不知道現在在哪個遊戲階段 | 固定顯示狀態列 / 按鈕組根據階段切換 |
| 按鈕灰化無說明 | 玩家不知道為什麼 Double 不能用 | Disabled 按鈕 hover 顯示 Tooltip |
| 派彩動畫太長 | 玩家等待焦慮 | 派彩動畫最長 2s，可點擊跳過 |
| Mobile 底部被遮擋 | 決策按鈕被虛擬鍵盤或系統手勢遮蓋 | Fixed bottom + safe-area-inset-bottom |
| 倒數超時行為不明 | 玩家不知道超時會自動做什麼 | 倒數條旁顯示「超時將自動：[行為]」|
| Split 後操作混亂 | 玩家不清楚在操作哪一手 | 高亮當前決策手 + 明確的切換動畫 |
| 重複下注 | 玩家不小心多點一次籌碼 | 每次點擊有音效 + 視覺累積確認 |
| 結果看不清 | 勝負 Toast 與背景撞色 | 半透明遮罩 + 強對比文字 |

## 六、視訊直播（Live Dealer）介面設計專章

### 6.1 直播畫面構成概念

視訊直播 Blackjack 介面由兩層疊合構成：

```
┌─────────────────────────────────────────┐
│         Layer 2: UI Overlay（前景）       │  ← 玩家互動控制、籌碼、按鈕、計分
├─────────────────────────────────────────┤
│         Layer 1: Live Video（背景）       │  ← 真人荷官、牌桌、發牌機、實體牌卡
└─────────────────────────────────────────┘

設計原則：
  - UI Overlay 不得遮擋荷官臉部、手部動作、牌卡
  - 透明度與模糊層需保留直播畫面可讀性
  - UI 動畫不干擾玩家視線對荷官動作的追蹤
```

---

| 牌靴鏡頭 | 側角 | 洗牌、補牌動作 | 局間過場 |

### 6.3 空間分區規劃（直播畫面熱區）

```

│ ┌──────────────┐                  ┌────────────────┐ │
│ │  ZONE A      │                  │   ZONE B       │ │
│ │  荷官臉部區   │  ◄ 絕對禁止遮擋  │   路紙 /        │ │
│ │  (上方 20%)  │                  │   遊戲資訊欄    │ │
│ └──────────────┘                  └────────────────┘ │
│ ┌────────────────────────────────────────────────────┐│
│ │                   ZONE C                           ││
│ │              荷官手牌展示區                          ││
│ │          (畫面中上 25–50% 水平帶)                   ││
│ │   ◄ 禁止常態 UI 元件，允許短暫結果 Overlay           ││
│ └────────────────────────────────────────────────────┘│
│ ┌────────────────────────────────────────────────────┐│
│ │                   ZONE D                           ││
│ │              玩家牌區 + 計分 Overlay                 ││
│ │          (畫面中央 50–70% 水平帶)                   ││
│ │   ◄ 手牌點數、BJ/BUST 標記疊加於實體牌上方           ││
│ └────────────────────────────────────────────────────┘│
│ ┌────────────────────────────────────────────────────┐│
│ │                   ZONE E                           ││
│ │              玩家控制區（UI 主區）                   ││
│ │          (畫面下方 30%)                             ││
│ │   投注區 | 籌碼面板 | 決策按鈕 | 倒數計時器          ││
│ └────────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

#### 各 Zone 設計規則

| Zone | 名稱 | UI 疊加規則 |
|---|---|---|
| A | 荷官臉部區 | **禁止任何 UI 元件**。最多允許極小角落的房間名稱標籤 |
| B | 遊戲資訊欄 | 允許路紙、RTP 說明、遊戲名稱、限注資訊（半透明背景）|
| C | 荷官牌展示區 | 禁止常態 UI；僅允許結算時短暫的點數 Overlay（1–2s 自動消失）|
| D | 玩家牌區 | 允許點數標籤、BJ/BUST 標記疊加；標籤需帶陰影確保可讀性 |
| E | 玩家控制區 | UI 主要操作區，允許全部互動元件 |

### 6.4 UI Overlay 設計規範

#### 透明度層次系統

```
Level 0: 直播畫面（100% 不透明，最底層）
Level 1: 半透明遮罩（Zone E 底部控制區，建議 rgba(0,0,0,0.55–0.70)）
Level 2: 牌面資訊標籤（Zone D，帶描邊陰影，確保任何牌桌背景下可讀）
Level 3: Toast / 結果通知（短暫全寬，消失後恢復直播）
Level 4: 重要彈窗（保險詢問、確認對話框，帶較深遮罩）
```

#### Zone E 控制列配置（由下往上）

```
┌─────────────────────────────────────────────┐
│  [ Split ]  [ Double ]  [ Hit ]  [ Stand ]   │ ← Row 1: 決策按鈕
├─────────────────────────────────────────────┤
│  [投注區: 主注] [Perfect Pairs] [21+3] [HOT3]│ ← Row 2: 投注區
├─────────────────────────────────────────────┤
│  [1] [5] [25] [100] [500]  [清除] [重注]     │ ← Row 3: 籌碼面板
├─────────────────────────────────────────────┤
│  ████████████░░░░  12s      [Chat] [設定]    │ ← Row 4: 倒數 + 工具列
└─────────────────────────────────────────────┘
  ↑ safe-area-inset-bottom（Mobile 安全區）
```

### 6.6 牌卡辨識輔助 UI

視訊串流中牌面可能因壓縮或角度不清晰，需 UI 輔助確認：

```
牌面點數 Overlay 規範：
  - 位置：疊加於對應實體牌的正上方（跟隨牌的位置）
  - 尺寸：小標籤，不蓋住牌面超過 30%
  - 樣式：帶黑色描邊的白色粗體字
  - 顯示時機：發牌後立即出現，翻牌動畫後更新
  - Soft Hand：顯示雙值，例「7│17」

荷官暗牌顯示：
  - 翻牌前：「?」佔位符（灰色）
  - 翻牌瞬間：「?」→ 實際點數的翻轉動畫（300–400ms）
```

### 6.8 直播品質 × UI 設計應對策略

| 直播問題 | UI 應對方案 |
|---|---|
| 低解析度 / 壓縮失真 | 強制顯示牌面點數 Overlay，不依賴玩家自行辨識 |
| 高延遲（>500ms） | 倒數計時不依賴伺服器時間，客戶端自走，最終由伺服器校正 |
| 鏡頭抖動 | UI 元件固定定位，不跟隨鏡頭晃動 |
| 換荷官過場 | 進入 IDLE 狀態，播放 Loading 動畫覆蓋過場 |
| 斷線重連 | 重連後自動恢復當前遊戲狀態，Toast 提示「重連成功」|
| 多桌切換 | 各桌 UI 狀態獨立，切換時保留投注不中斷 |

## 七、回答行為規範

### 收到需求時，先確認以下面向再給答案：

1. **目標版本**：標準 BJ / Infinite / Spanish 21 / 其他變異？
2. **問題類型**：規則詢問 / 策略計算 / UI 設計 / UX 流程 / 賠率分析 / 直播介面？
3. **平台**：Web Desktop / Mobile App（直式/橫式）/ 真實荷官（Live Dealer）串流介面？
4. **直播相關**：主播站位方向？鏡頭俯角？哪個 Zone 有遮擋問題？Mobile 橫式或直式？
5. **玩家視角**：新手友善 or 老手效率優先？
6. **具體情境**：哪個遊戲階段？什麼樣的手牌 / 操作場景？

### 輸出格式要求

- 規則說明：表格 + 流程圖（ASCII 或 Mermaid）
- 策略建議：對照表 + 情境解說
- UI/UX 設計：分階段列出元件狀態 + 設計原則 + 反例對照
- 賠率計算：公式 + 數值 + 實際例子

## 八、賠率與莊家優勢參考
