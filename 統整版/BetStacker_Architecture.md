## 二、User Flow 核心路徑

```
開始新局
│
▼
[Phase 1 · Betting — 下注階段]
玩家放置主注（必填）+ 邊注（選填）+ 後面跟注（選填）
系統自動扣除 10% Bet Stacker 費用並顯示提示
│
▼
[Phase 2 · Bet Stacker — 隨機抽獎]
系統隨機抽取：獎勵元素 Rank + 獎金乘數（25/50/75/100%）
動畫展示結果 → 顯示本局獎金價值（主注 × 乘數）
│
▼
[Phase 3 · Deal — 發牌]
莊家發初始兩張牌（玩家 2 明牌，莊家 1 明 1 暗）
比對玩家手牌 Rank ↔ 獎勵元素
  ├── 0 張符合 → 主注不變
  ├── 1 張符合 → 主注 + 1× 獎金價值
  └── 2 張符合 → 主注 + 2× 獎金價值
│
▼
[Phase 4 · Blackjack Check]
  ├── 玩家 BJ + 莊家 BJ → Push（退還疊加後全額注碼）
  ├── 玩家 BJ 且莊家無 → 玩家勝 6:5
  ├── 莊家 BJ 且玩家無 → 玩家負
  └── 雙方無 BJ → 進入操作階段
│
▼
[Phase 5 · Action — 玩家操作]
  ├── Cashout  → 確認 Dialog → 立即結算，本局結束
  ├── Split    → 分成兩手，各繼承新注碼，分別操作
  ├── Double   → 以新注碼為基礎加倍，只再抽一張牌
  ├── Hit      → 再抽一張牌；爆牌 → 玩家負
  └── Stand    → 鎖定手牌，進入莊家操作
│
▼
[Phase 6 · Dealer — 莊家操作]
翻開底牌 → ≤16 Hit / Soft 17+ Stand → 直到停牌或爆牌
│
▼
[Phase 7 · Settlement — 結算]
比較最終點數，派彩或收注
邊注各自獨立結算（不受主注勝負影響）
│
▼
新局開始 ↺
```

## 四、MVP 必要頁面（P0 + P1）

### P0 核心 — 玩家能完成完整一局的最小集合

| 優先 | 頁面 | 關鍵原因 |
|---|---|---|
| P0 | 主遊戲桌 Game Table | 所有 Phase 的容器，缺少無法遊戲 |
| P0 | 下注介面 | 每局起點，主注為強制操作 |
| P0 | Bet Stacker Phase | **本遊戲差異化核心特色**，缺少即失去產品定義 |
| P0 | 發牌 + 疊注判定顯示 | 注碼變化必須清晰呈現 |
| P0 | 玩家操作列 | 玩家決策主介面（5 種操作） |
| P0 | 套現確認 Dialog | PRD 明確要求防誤觸；Cashout 為核心功能 |
| P0 | 結算顯示 | 局末必要，玩家需知道輸贏與金額 |
| P0 | 餘額 HUD | 玩家需知道餘額才能做下注決策 |

### P1 基本 — 產品上線必要但不阻擋核心流程

| 優先 | 頁面 | 關鍵原因 |
|---|---|---|
| P1 | 大廳 / 選桌 Lobby | 遊戲入口；單桌版可簡化 |
| P1 | 斷線重連 Reconnect | PRD §9 明確列為邊界條件，需要處理 |

### P2 完善 — Post-MVP 迭代

| 頁面 | 說明 |
|---|---|
| 後面跟注介面 | 選填功能，可後續迭代 |
| 遊戲說明 / 規則頁 | 可先以 tooltip/popup 替代獨立頁 |
| 遊戲歷史 History | 體驗加分，非遊玩流程必要 |

## 五、設計重點提醒

1. **Bet Stacker Phase 動畫** — 直接影響核心體驗，P0 投入設計資源（轉盤/抽卡動效）
2. **注碼顯示格式** — 疊加前後需統一規範：`$100 → $250（+$150 Stacked）`
3. **Cashout 即時金額** — 動態更新值，UI 需支援數字快速滾動動畫
4. **操作按鈕狀態** — Double/Split 有時機限制，需有清晰的 Enabled/Disabled 視覺差異
5. **邊注獨立結算** — 結算畫面需同時呈現主注 + 最多 4 種邊注結果，需規劃版面空間
