## 零、Wireframe 工作前的「先問後答」

**每次收到需求時，先提出 4～7 個精準問題，再給線框方案。** 缺少這些資訊就盲畫，必返工。

### 必問面向

1. **玩法版本**：標準（抽傭）／免傭（Commission-Free）／龍寶 Dragon Bonus／Lightning／Speed／Squeeze／多桌？
2. **目標畫面 / 階段**：投注中 / 發牌中 / 補牌（第三張規則）/ 結算 / 局間 / 哪個狀態？
3. **平台與版型**：Mobile 直式 / Mobile 橫式 / Web Desktop？是否需 responsive 對照？
4. **直播層級**：是否真人荷官視訊？鏡頭俯角？哪個 Zone 有遮擋風險？
5. **路紙需求**：要顯示哪幾種路（大路 / 大眼仔 / 小路 / 曱甴路 / 珠盤路 / 問路預測）？
6. **下注區**：只有莊閒和，還是含對子 / 龍寶 / Super 6 / 完美對子等邊注？
7. **保真度**：低保真線框（灰階方塊）/ 中保真（含元件狀態）/ 還是流程＋版型規格文件？

> 玩家視角補問：新手友善（路紙說明、賠率提示常駐）還是老手效率優先（密集資訊、快速重注）？

## 一、百家樂規則核心（線框前必須熟知）

線框師不需要算牌，但必須懂規則，否則畫不出正確的階段與資訊呈現。

### 1.1 基本玩法

```
百家樂只比「莊（Banker）」與「閒（Player）」兩家誰的點數接近 9。
玩家下注：莊贏 / 閒贏 / 和（Tie），可加下對子與邊注。
玩家「不持牌、不決策」——所有補牌依固定規則自動執行。
```

### 1.2 牌值計算

| 牌面 | 點數 |
|---|---|
| A | 1 |
| 2–9 | 面值 |
| 10 / J / Q / K | 0 |

- 兩張牌相加 **取個位數**（例：7+8=15 → 5 點）。
- 8 或 9 點為 **例牌（Natural）**，立即停牌不補牌。

### 1.3 第三張牌規則（補牌邏輯，線框需畫出「補牌中」狀態）

```
閒家：
  前兩張 0–5 → 補一張；6–7 → 停牌；8–9 → 例牌停牌

莊家：依閒家第三張牌點數，套用莊家補牌表（固定規則）
  莊 0–2 → 補
  莊 3 → 閒第三張非 8 才補
  莊 4 → 閒第三張 2–7 才補
  莊 5 → 閒第三張 4–7 才補
  莊 6 → 閒第三張 6–7 才補
  莊 7 → 停牌
```

> 線框重點：補牌是**系統自動**，UI 無決策按鈕，但需有清楚的「發牌中／補牌中」動態狀態與牌位佔位。

### 1.4 下注區與賠率（線框需依玩法切換）

| 下注項目 | 標準賠率 | 備註 |
|---|---|---|
| 閒 Player | 1:1 | — |
| 莊 Banker | 1:1（抽 5% 傭金） | 免傭版改 1:1，但莊 6 點贏只賠 1:2 |
| 和 Tie | 8:1（部分 9:1） | — |
| 莊對 / 閒對 Pair | 11:1 | 邊注 |
| 完美對子 Perfect Pair | 25:1 | 邊注 |
| 龍寶 Dragon Bonus | 依差距 1:1～30:1 | 邊注 |
| Super 6（免傭莊 6 贏） | 12:1 | 免傭版專屬 |

## 二、遊戲流程狀態機（線框分鏡的骨架）

每個狀態 = 一張線框；用此圖決定要畫幾張、各畫面元件鎖定關係。

```
[IDLE / 局間]
  │  顯示上局結果、路紙、限紅、下一局倒數
  ▼
[BETTING 投注中]            ← 籌碼面板 + 莊閒和 + 邊注區「亮起可點」
  │  倒數計時；可清除 / 重注 / 加注
  ▼
[NO_MORE_BETS 停止投注]     ← 所有下注區鎖定，視覺灰化 + 「停止下注」提示
  │
  ▼
[DEALING 發牌中]           ← 閒兩張、莊兩張依序入位；點數動態顯示
  │
  ├─ 例牌(8/9) ─▶ [SETTLEMENT]
  │
  ▼
[THIRD_CARD 補牌中]        ← 依規則自動補第三張（閒先、莊後）；Squeeze 版有搓牌動畫
  │
  ▼
[SETTLEMENT 結算]          ← 判定莊/閒/和；派彩動畫；路紙即時更新
  │
  ▼
[IDLE / 局間]
```

### 階段 × UI 狀態對應表

| 階段 | 主要元件 | 可互動 | 鎖定 |
|---|---|---|---|
| IDLE | 路紙、結果、倒數 | 預下注（部分桌規） | 下注區 |
| BETTING | 籌碼面板、莊閒和、邊注、倒數 | 籌碼、下注區、清除、重注 | — |
| NO_MORE_BETS | 灰化下注區、提示條 | 無 | 全部下注 |
| DEALING | 牌位、點數標籤 | 無（等待） | 全部 |
| THIRD_CARD | 牌位、補牌動畫 | 無（Squeeze 版可搓牌） | 全部 |
| SETTLEMENT | 結果 Toast、派彩、路紙更新 | 重注、續玩 | 下注區（短暫） |

## 三、畫面分區（Zone）規劃 — 直播百家樂

直播百家樂由兩層疊合：**底層真人視訊 + 前景 UI Overlay**。
核心鐵律：**UI 不得遮擋荷官臉部、手部發牌動作、與實體牌卡。**

```
┌──────────────────────────────────────────────┐
│ ZONE A：頂部資訊列                              │ ← 桌名 / 限紅 / 餘額 / 設定 / 離開
├──────────────────────────────────────────────┤
│                                              │
│ ZONE B：真人荷官視訊                            │ ← 荷官 + 牌靴 + 實體牌（俯角 30–45°）
│   ◄ 絕對禁止常態 UI 遮擋荷官臉與手               │
│                                              │
│   ┌────────────────────────────────────┐    │
│   │ ZONE C：牌區疊層                      │    │ ← 閒/莊點數標籤、例牌標記
│   │  [閒 PLAYER]  vs  [莊 BANKER]        │    │   疊於實體牌上方
│   └────────────────────────────────────┘    │
│                                              │
├──────────────────────────────────────────────┤
│ ZONE D：路紙區                                 │ ← 大路 + 下三路 + 珠盤路 + 問路
├──────────────────────────────────────────────┤
│ ZONE E：下注區（莊 / 閒 / 和 / 對子 / 邊注）     │ ← 主互動熱區
├──────────────────────────────────────────────┤
│ ZONE F：籌碼面板 + 操作 + 倒數                  │ ← 籌碼 / 清除 / 重注 / 倒數條
└──────────────────────────────────────────────┘
│ ZONE G：結算列 + 餘額顯示                       │ ← 結算列 + 餘額顯示 + 限紅 + 桌名
└──────────────────────────────────────────────┘


```

### 各 Zone 設計規則

| Zone | 名稱 | 線框規則 |
|---|---|---|
| A | 頂部資訊列 | 半透明深色底；左桌名、右餘額/設定。高度精簡，不壓縮視訊。 |
| B | 荷官視訊 | **禁止常態 UI**。視訊為最底層；UI 僅可在四周與下方疊放。 |
| C | 牌區疊層 | 僅放點數標籤、Natural/結果標記；帶描邊陰影確保任何背景可讀；結算後短暫顯示。 |
| D | 路紙區 | 可收合；Mobile 直式預設顯示大路 + 縮圖下三路，點擊展開全路。 |
| E | 下注區 | 主互動區；莊閒和三大區 + 對子/邊注；狀態隨階段切換（亮/鎖/中獎高亮）。 |
| F | 操作區 | 籌碼、清除、重注、倒數；Mobile 固定底部 + `safe-area-inset-bottom`。 |
| G | 結算列 | 結算結果、餘額變動；Mobile 固定底部 。 |

## 四、Mobile 直式線框（主力版型）

```
┌─────────────────┐  ← 寬 375–430px
│ A 桌名｜限紅｜⚙  │  ← 頂列 44px
├─────────────────┤
│                 │
│   B 荷官視訊     │  ← 視訊 16:9，約螢幕 38–45%
│   (16:9)        │
│  ┌───────────┐  │
│  │C 閒 vs 莊  │  │  ← 點數標籤疊層
│  └───────────┘  │
├─────────────────┤
│ D 大路  [展開▼]  │  ← 路紙縮圖列 56–72px，可展開
├─────────────────┤
│   E 下注區       │
│  ┌─────┬─────┐  │
│  │ 閒  │ 莊  │  │  ← 莊閒左右，和置中或上方
│  └─────┴─────┘  │
│  [  和 TIE  ]   │
│  [閒對][和對][莊對]│  ← 邊注一列
├─────────────────┤
│ F [1][5][25][100]│  ← 籌碼面板
│  [清除][重注] 12s │  ← 操作 + 倒數
└─────────────────┘  ← safe-area
```

### Mobile 直式重點

- **視訊不被擠壓**：路紙與下注區可滑動／收合，但視訊區保持 16:9。
- **拇指熱區**：籌碼與莊閒和置於螢幕下半，符合單手操作。
- **路紙折疊**：預設只露大路，點「展開」以半屏 Sheet 顯示全部下三路 + 珠盤路。
- **倒數可見**：倒數條置於下注區上緣或籌碼列右側，永遠在視線內。

## 六、下注區線框規範（Zone E 細節）

### 6.1 三大主注佈局原則

```
實體百家樂桌佈局慣例：  閒(Player) ← 中(Tie) → 莊(Banker)
線框沿用此左右關係，降低老手認知負荷。
和(Tie) 通常置中或在莊閒之間上方，視覺權重低於莊閒。
```

### 6.2 下注區狀態（每區皆需畫出）

| 狀態 | 視覺 | 出現時機 |
|---|---|---|
| Default | 清晰邊框、賠率標示 | BETTING |
| Hover（Desktop） | 高亮邊框 + Tooltip 賠率 | 滑過 |
| Active / 已下注 | 疊加籌碼 + 金額標籤 | 點擊後 |
| Locked | 灰化、降透明度 | NO_MORE_BETS 之後 |
| Win 中獎 | 金光高亮 + 派彩數字 | SETTLEMENT |
| Lose | 暗化 + 籌碼飛出 | SETTLEMENT |

### 6.3 籌碼面板與操作

```
1. 面額由小到大排列，常用面額居中或預設選中
2. 選中籌碼明確高亮
3. 點下注區 = 疊加選中籌碼；長按/右鍵 = 移除一枚
4. [清除] 一鍵清空；[重注] 帶回上局；[雙倍] 全下注 ×2
5. 每次下注有音效 + 視覺累積，防誤觸重複下注
```

## 七、路紙系統線框（百家樂靈魂，務必正確）

百家樂路紙是玩家核心需求，線框必須畫對結構，不可只放一塊「路紙」灰框。

### 7.1 五路結構

```
珠盤路（Bead Plate）   ← 逐局結果，6 列 × N 欄，紅=莊 藍=閒 綠=和，角點標對子
大路（Big Road）       ← 莊閒連勝成龍，遇和畫斜線/數字，6 列換欄
─── 以下為大路衍生的「下三路」，預測用 ───
大眼仔（Big Eye Boy）  ← 看「規律性」，紅/藍圈
小路（Small Road）     ← 看「規律性」，紅/藍實心點
曱甴路（Cockroach）    ← 看「規律性」，紅/藍斜線
問路（Ask Road）       ← 「若下局開莊 / 開閒」三路會如何，預測提示
```

### 7.2 線框佈局建議

```
┌──────────────────────────────────────┐
│ 珠盤路（左半）        │ 大路（右半）     │  ← 上排
├───────────────────────┴────────────────┤
│ 大眼仔  │  小路        │  曱甴路         │  ← 下三路一列
├──────────────────────────────────────┤
│ 問路：[莊?] [閒?]   統計：莊12 閒9 和2  │  ← 問路 + 計數
└──────────────────────────────────────┘
```

- **Mobile 直式**：預設只露「大路」橫向縮圖，點擊展開半屏顯示全部。
- **Desktop**：右欄常駐全展開。
- **顏色規範**：莊=紅、閒=藍、和=綠；對子以小角點（莊對紅點、閒對藍點）標於格角。

## 八、結算與回饋動畫（線框需標注時長與層級）

| 事件 | 動畫 | 時長 | 線框標注 |
|---|---|---|---|
| 發牌 | 滑入 + 翻牌 | 300–500ms/張 | 牌位佔位 + 入場方向 |
| Squeeze 搓牌 | 慢揭 + 放大 | 依玩家操作 | 僅 Squeeze 版，標互動區 |
| 例牌 Natural | 金字標記 | 600ms | Zone C 疊層 |
| 莊/閒勝 | 對應色高亮 + 派彩 | 1–2s | 中獎下注區發光 |
| 和 Tie | 綠光 + 退莊閒注 | 800ms | 標注莊閒「退注」非輸 |
| 派彩 | 籌碼飛入 + 數字滾動 | 1–2s，可點擊跳過 | 防等待焦慮 |
| 路紙更新 | 新點滑入 | 300ms | 結算後即時 |

> 倒數計時三段色：>50% 綠、20–50% 黃、<20% 紅閃。投注超時 → 不下注或保留上注（依桌規），需於線框旁標注超時行為。

## 十、常見 Wireframe 坑洞 × 避坑

| 坑洞 | 後果 | 避坑 |
|---|---|---|
| UI 遮擋荷官臉/手 | 直播沈浸感破壞、違規 | Zone B 留淨空，UI 只在四周與下方 |
| 路紙畫成單一灰框 | 工程無法實作、玩家無法用 | 五路結構各自獨立元件 + 顏色規範 |
| 莊閒左右顛倒 | 老手認知衝突 | 沿用實體桌「閒左莊右」 |
| 和局畫成「莊閒皆輸」 | 派彩邏輯錯 | 和局莊閒為「退注」，需明確標注 |
| 補牌階段無狀態 | 玩家以為卡頓 | 畫出 DEALING / THIRD_CARD 動態佔位 |
| 免傭/抽傭賠率混用 | 賠率標示錯 | 依玩法版本切換賠率文案（莊 6 點 1:2 等） |
| Mobile 視訊被擠壓 | 看不清荷官 | 視訊鎖 16:9，路紙/下注區用收合與滑動 |
| 倒數結束行為不明 | 玩家錯失下注 | 倒數旁標注超時行為 |
| 底部被手勢條遮擋 | 按鈕點不到 | `safe-area-inset-bottom` |
| 邊注擠爆主注區 | 誤觸、視覺雜亂 | 邊注獨立分列，視覺權重低於莊閒和 |

### 11b.1 老虎百家樂 Tiger Baccarat（Evolution，免佣制）

**玩法特色**：本質是免佣百家樂（莊 6 點贏只賠 1:2），主打圍繞「莊 6」的一整組老虎邊注。

**新增下注區與賠率（Evolution 版，依平台校準）**

| 下注項目 | 賠率 | 命中條件 |
|---|---|---|
| Tiger（老虎） | 2 張 12:1 / 3 張 20:1 | 莊以 6 點贏 |
| Small Tiger（小虎） | 22:1 | 莊「兩張」6 點贏 |
| Big Tiger（大虎） | 50:1（部分 55:1） | 莊「三張」6 點贏 |
| Tiger Tie（虎和） | 45:1（部分 35:1） | 和局且總點為 6 |
| Tiger Pair（虎對） | 單對 4:1 / 雙對 25:1 / 對牌相同 100:1 | 首兩張成對（依對子型態分級） |

**線框需調整處**
- Zone E 需一次容納 **莊/閒/和 + 5 個老虎邊注**；建議把老虎邊注獨立成一列或可展開群組，視覺權重低於三大主注。
- 莊賠率文案改「1:1（莊 6 點 1:2）」，與標準抽傭版明確區分。
- 結算高亮要能同時打亮「主注 + 命中的老虎邊注」（可能多區同時中獎）。

**避坑**
- 老虎邊注一多容易擠爆主注區 → 用分列/收合群組，別和莊閒和平鋪。
- 5 個邊注賠率差異大 → 每區常駐賠率標示，勿只靠 Tooltip（Mobile 無 hover）。

---

### 11b.2 Peek Baccarat（Evolution，免佣 + 例牌倍率）

**玩法特色**：免佣百家樂，荷官在關鍵時「偷看（Peek）」最後一張，對「以例牌（8/9）獲勝」提供大額倍率彩金。

**新增下注區與賠率**
- 主注莊/閒/和照舊（莊 6 點 1:2）。
- 倍率不是固定賠率，而是**該局隨機生成的例牌獲勝倍率**（依平台，最高可達數十至上百倍）→ 屬「動態賠率」，需以資料驅動顯示。

**線框需調整處**
- 新增 **偷看倍率揭示疊層**：發牌/補牌階段以慢揭動畫顯示是否觸發倍率，疊於 Zone C 牌區上方，帶描邊確保可讀。
- 結算需清楚呈現「基礎彩金 × 倍率」的組成，避免玩家看不懂為何金額暴增。

**避坑**
- 倍率只在「例牌獲勝」時生效 → 線框要標注觸發條件，別讓玩家誤以為每局都有。
- 慢揭動畫勿遮擋荷官手部（Zone B 鐵律）。

---

### 11b.3 XXXtreme Lightning Baccarat（Evolution，倍率版 Plus）

**玩法特色**：Lightning 家族最強版。每局隨機 **4～8 張倍率牌（2x/3x/4x/5x/8x/10x）**，另有 **Lightning Chip Box** 落在莊/閒/和/對子區再加乘。收取 **50% Lightning Fee**。

**賠付重點**
- 閒贏 1:1、莊贏 0.97:1（已內含調整）、和 8:1，全部再乘上命中的倍率。
- 單手最高 1000:1（10x×10x×10x）；和局極端可達天文倍率，故設**單局派彩上限**。

**線框需調整處**
- **局前倍率牌展示區**：顯示本局隨機倍率牌與數值。
- **Chip Box 疊層**：在下注區上標出哪幾區被 Chip Box 命中及其倍率（下注區需要「倍率徽章」狀態）。
- **Fee 明示**：下注列旁常駐「+50% Fee」與「實際扣款 = 下注 ×1.5」預覽。
- 結算需展示「倍率鏈」（多張倍率牌相乘）與封頂提示。

**避坑**
- 50% Fee 沒講清楚是最大客訴來源 → 下注確認前就要顯示總扣款。
- 倍率標記層級多（牌區倍率 + Chip Box 倍率）→ 用不同視覺語彙區分，別混成一團。
- 派彩上限要有明確 UI，避免玩家誤算期望。

> 一般版 **Lightning Baccarat** 為其簡化版：僅局前倍率牌（無 Chip Box），Fee 20%，其餘同第十一章「Lightning／倍率版」。

---

### 11b.4 EZ Baccarat（免佣）＋ 龍7 / 熊貓8

**玩法特色**：免佣百家樂。莊以「**三張牌總點 7**」獲勝時 → 主注莊注**推注（Push，不輸不贏）**，改由 Dragon 7 邊注結算。

**新增下注區與賠率**

| 下注項目 | 賠率 | 命中條件 |
|---|---|---|
| Dragon 7（龍 7） | 40:1 | 莊以「三張 7 點」獲勝 |
| Panda 8（熊貓 8） | 25:1 | 閒以「三張 8 點」獲勝 |

**線框需調整處**
- 新增龍7 / 熊貓8 兩個邊注區（可用龍／熊貓圖示強化辨識）。
- **新增「Push 推注」結算狀態**：莊三張 7 時，莊注既非 Win 也非 Lose，需第七種下注區狀態（`_Push`：籌碼原位返還 + 「退回」標記）。
- 結算文案要能同時表達「莊注推注 + 龍7 中獎」。

**避坑**
- 漏掉 Push 狀態 → 玩家以為系統吞注。務必獨立標注。
- 龍7/熊貓8 是「特定三張牌型」才中 → 賠率旁加簡短命中條件提示。

---

### 11b.5 Golden Wealth Baccarat（Pragmatic Play，倍率版）

**玩法特色**：PP 版倍率百家樂。每局隨機「**金卡**」帶 2x～8x 倍率，命中即加乘；並常搭配莊/閒對子等邊注。

**線框需調整處**
- 同 Lightning 家族：需**金卡倍率展示區** + 下注區/牌區倍率徽章。
- 依平台是否收 Fee 調整下注列提示（PP 版倍率機制與 Evolution 不完全相同，實作前對規則表）。

**避坑**
- 別直接套用 Evolution 的 Fee 文案 → 供應商機制不同，賠率/Fee 一律走資料驅動。

---

### 11b.6 免佣家族：Super 6 / Mega 6 / Fortune / No Commission

**玩法特色**：莊贏一律 1:1（不抽 5% 傭金），但**莊以 6 點贏時特殊處理**。

| 子版本 | 莊 6 點贏處理 | 額外下注區 |
|---|---|---|
| Super 6 / No Commission | 只賠 1:2 | 常配 Super 6 邊注（莊 6 贏 12:1） |
| Mega 6 | 主注照免佣，另設 Mega 6 邊注 | 莊 6 贏：2 張 12:1 / 3 張 20:1 |
| Fortune 6 | 概念同上，命名不同 | 依平台 |

**線框需調整處**
- 莊注賠率文案務必標「莊 6 點 1:2」並在結算時明示減額。
- 若含 Super6/Mega6 邊注 → 獨立一格，賠率常駐。

**避坑**
- 免佣 vs 抽傭賠率混用是經典錯（見第十章）→ 依玩法版本切換文案，勿共用同一份賠率字串。

---

### 11b.7 First Person Baccarat（RNG，無真人荷官）★版型大改

**玩法特色**：非直播，**電腦 3D 牌桌 + RNG 發牌**，無荷官視訊。常附「Go Live」切換到真人桌、統計面板、Autoplay。

**線框需調整處（與直播版最大差異）**
- **Zone B 不再是 16:9 視訊**：改為 3D 牌桌畫布，可自由運用整塊主區，不受「不遮荷官臉/手」鐵律限制 → UI 可更貼近牌桌。
- 新增 **「Go Live」入口**（切換到對應真人桌）。
- 通常保留完整路紙 + 統計 + Autoplay/快速下注，適合桌面與長時操作。
- 動畫可調速甚至跳過（RNG 無真人節奏限制）。

**避坑**
- 沿用直播版「Zone B 淨空」規範會浪費空間 → RNG 版應重排主區。
- 別忘了「Go Live / 回 RNG」的雙向切換入口與狀態保留。

---

### 11b.8 Salon Privé / Control Squeeze / Speed Squeeze（VIP 搓牌）

**玩法特色**：專屬高限紅桌；玩家可**控制發牌節奏、親自搓牌（Squeeze）**，強沈浸感。

**線框需調整處**
- 在第十一章 Squeeze 基礎上，加 **控牌操作區**（發牌/揭牌節奏由玩家觸發）。
- 高限紅 → 籌碼面額表上調、限紅資訊更顯眼。
- 專屬視覺主題（VIP 質感），但仍守 Zone B 荷官淨空鐵律。

**避坑**
- 搓牌互動區勿與下注區重疊誤觸。
- 限紅級距大 → 下注確認/二次確認防手滑重注。

## 十二、輸出格式要求

收到需求並問清後，依此結構輸出線框方案：

1. **狀態分鏡清單**：列出要畫哪幾張（依第二章狀態機）。
2. **ASCII 線框圖**：每個目標畫面用 ASCII 方塊呈現分區與元件位置。
3. **Zone 規則表**：各區互動 / 鎖定 / 疊層規則。
4. **元件狀態表**：下注區、籌碼、按鈕的各狀態。
5. **命名 + 標注**：依第九章命名，標出動畫時長、超時行為、賠率文案。
6. **避坑提醒**：點出此需求最可能踩的 2～3 個坑。

需要時可加：Mobile 直式 ↔ Desktop 響應式對照、Figma 元件層級建議。
