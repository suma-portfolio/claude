# Part A — 兩遊戲共通基礎（直播 × UI/UX × 線框）

> 以下規範**百家樂與 21 點通用**。各遊戲特有的狀態機、下注區、路紙、Split 見 Part B / Part C。

## A1 直播（Live Dealer）兩層疊合概念

視訊直播牌桌介面由兩層疊合構成：

```
┌─────────────────────────────────────────┐
│         Layer 2: UI Overlay（前景）       │  ← 玩家互動：籌碼、按鈕、計分、下注區
├─────────────────────────────────────────┤
│         Layer 1: Live Video（背景）       │  ← 真人荷官、牌桌、發牌機、實體牌卡
└─────────────────────────────────────────┘
```

**核心鐵律（兩遊戲皆適用）：**
- UI Overlay **不得遮擋荷官臉部、手部動作、實體牌卡**。
- 透明度與模糊層需保留直播畫面可讀性。
- UI 動畫不干擾玩家對荷官動作的視線追蹤。

> ⚠ 例外：**First Person / RNG 版**（電腦 3D 牌桌、無真人視訊）不受此鐵律限制，
> 主區可自由運用、重排（見 B8）。

## A2 攝影機構圖與角度

| 鏡頭類型           | 角度        | 用途             | 切換時機      |
| -------------- | --------- | -------------- | --------- |
| 主鏡頭（Wide）      | 俯角 30–45° | 全桌全覽（荷官 + 牌區）  | 預設常駐      |
| 特寫鏡頭（Close-up） | 俯角 60–80° | 牌面確認（翻牌、補牌、結算） | 自動切換      |
| 荷官正面鏡頭         | 水平視角      | 互動感、情境感        | IDLE 等待期  |
| 牌靴/搓牌鏡頭        | 側角        | 洗牌、補牌、Squeeze  | 局間過場 / 搓牌 |

## A3 畫面分區（Zone）通用原則

不論百家樂或 21 點，直播牌桌畫面都可抽象為「上：荷官與牌 / 中：資訊 / 下：玩家控制」三帶：

| 區帶  | 內容                    | UI 疊加規則                              |
| --- | --------------------- | ------------------------------------ |
| 頂部  | 桌名 / 限紅 / 餘額 / 設定 / 離開 | 半透明深色底，高度精簡，不壓縮視訊                    |
| 荷官區 | 荷官臉部 + 手部 + 牌靴         | **禁止任何常態 UI**，最多角落極小房間名稱標籤            |
| 牌區  | 荷官牌 / 玩家牌             | 僅點數/結果標籤疊層，帶描邊陰影；結算時短暫顯示（1–2s 自動消失）  |
| 資訊區 | 百家樂=路紙；21點=無（或統計）      | 半透明背景，可收合                            |
| 下注/控制 | 投注區 / 籌碼 / 按鈕 / 倒數     | UI 主操作區，允許全部互動元件                     |

> 各遊戲的完整 Zone 切分：百家樂見 B3（Zone A–G）、21點見 C7（Zone A–E）。

## A4 UI Overlay 透明度層次系統

```
Level 0: 直播畫面（100% 不透明，最底層）
Level 1: 半透明遮罩（底部控制區，建議 rgba(0,0,0,0.55–0.70)）
Level 2: 牌面資訊標籤（帶描邊陰影，確保任何牌桌背景下可讀）
Level 3: Toast / 結果通知（短暫全寬，消失後恢復直播）
Level 4: 重要彈窗（保險詢問、確認對話框、倍率揭示，帶較深遮罩）
```

## A5 籌碼面板 UX（兩遊戲共用）

```
1. 面額由小到大排列，最常用面額居中或預設選中
2. 選中籌碼有明確高亮效果
3. 點擊下注區 = 疊加選中籌碼
4. 長按 / 右鍵 = 移除一枚籌碼（降低誤操作損失）
5. [清除] 一鍵清空（需確認或快速動作）
6. [重注] 帶入上局注碼（節省操作）；[雙倍] 全下注 ×2（百家樂常見）
7. 每次下注有音效 + 視覺累積確認，防誤觸重複下注
```

## A6 計分 / 點數顯示

| 遊戲   | 顯示規則                                                 |
| ---- | ---------------------------------------------------- |
| 通用   | 點數標籤永遠跟隨手牌顯示（避免玩家視線移動）；帶描邊陰影確保可讀                     |
| 百家樂  | 閒/莊各顯示個位數點數；例牌（8/9 Natural）金字標記；暗牌未開以「?」佔位          |
| 21點  | Soft Hand 顯示雙值（例「7/17」）；Hard 單值；爆牌紅字 + BUST；BJ 金色高亮 |

## A7 倒數計時器 UX

| 遊戲   | 分段配色                                     | 超時行為                          |
| ---- | ---------------------------------------- | ----------------------------- |
| 百家樂  | >50% 綠 / 20–50% 黃 / <20% 紅閃              | 不下注或保留上注（依桌規），線框旁標注           |
| 21點  | >6s 綠 / <5s 紅閃（起始約 15s）                  | BETTING→自動重注或不下注；INSURANCE→自動 No；PLAYER_DECISION→自動 Stand（保護玩家） |

- 計時器位置：靠近當前需互動的元件（不讓視線游移）。
- Mobile：置頂或按鈕上方；下注區上緣或籌碼列右側，永遠在視線內。
- 高延遲（>500ms）：倒數不依賴伺服器時間，客戶端自走，最終由伺服器校正。

## A9 Mobile / Desktop 版型通則

- **視訊鎖 16:9，不被擠壓**：資訊區與下注區用收合/滑動釋放空間。
- **拇指熱區**：籌碼與主下注/決策鈕置於螢幕下半，符合單手操作。
- **底部安全區**：固定底部控制列必用 `safe-area-inset-bottom`，避免被手勢條/虛擬鍵盤遮擋。
- **Desktop**：空間足 → 資訊區（路紙/統計）常駐側欄；支援 hover 高亮與賠率 Tooltip。
- **Mobile 無 hover**：賠率必須常駐標示，不可只靠 Tooltip。

## A10 直播品質 × UI 應對策略

| 直播問題        | UI 應對方案                       |
| ----------- | ----------------------------- |
| 低解析度 / 壓縮失真 | 強制顯示牌面點數 Overlay，不依賴玩家自行辨識    |
| 高延遲（>500ms） | 倒數客戶端自走，伺服器最終校正               |
| 鏡頭抖動        | UI 元件固定定位，不跟隨鏡頭晃動            |
| 換荷官過場       | 進入 IDLE，播放 Loading 動畫覆蓋       |
| 斷線重連        | 自動恢復當前遊戲狀態，Toast 提示「重連成功」     |
| 多桌切換        | 各桌 UI 狀態獨立，切換保留投注不中斷          |

## A11 UX 核心法則

```
UI = 玩家看到什麼、能點什麼
UX = 玩家在整個流程中的感受與決策效率

博弈牌桌的 UX 目標：
  1. 降低認知負荷（不讓玩家迷失在哪個階段）
  2. 加速決策循環（減少操作步驟）
  3. 強化即時回饋（每個動作有明確視覺/聽覺響應）
  4. 建立情緒張力（贏的爽感、輸的不甘 → 繼續遊玩）
```

# Part B — 百家樂（Baccarat）模組

> 角色心法：線框師不需要算牌，但必須懂規則，否則畫不出正確的階段與資訊呈現。

## B1 規則核心（線框前必須熟知）

### B1.1 基本玩法
```
只比「莊 Banker」與「閒 Player」誰的點數接近 9。
玩家下注：莊贏 / 閒贏 / 和（Tie），可加對子與邊注。
玩家「不持牌、不決策」——所有補牌依固定規則自動執行。
```

### B1.2 牌值計算
| 牌面             | 點數 |
| -------------- | -- |
| A              | 1  |
| 2–9            | 面值 |
| 10 / J / Q / K | 0  |

- 兩張相加 **取個位數**（例 7+8=15 → 5 點）。
- 8 或 9 點為 **例牌（Natural）**，立即停牌不補牌。

### B1.3 第三張牌規則（線框需畫出「補牌中」狀態）
```
閒家：前兩張 0–5 → 補；6–7 → 停；8–9 → 例牌停
莊家：依閒家第三張牌點數，套用固定補牌表
  莊 0–2 → 補
  莊 3 → 閒第三張非 8 才補
  莊 4 → 閒第三張 2–7 才補
  莊 5 → 閒第三張 4–7 才補
  莊 6 → 閒第三張 6–7 才補
  莊 7 → 停
```
> 補牌是**系統自動**，UI 無決策按鈕，但需清楚的「發牌中／補牌中」動態狀態與牌位佔位。

# Part C — 21點（Blackjack）模組

> 與百家樂最大差異：**玩家需持牌並主動決策**（Hit/Stand/Double/Split/…），
> 故有「決策按鈕組」與「Split 多手牌版型」，且策略是核心價值。

## C1 規則核心

### C1.1 標準流程（State Machine）
```
[IDLE] → [BETTING] 玩家下主注+可選邊注
  → [DEALING] 玩家兩明牌；荷官一明一暗
     ├ 荷官明牌=A → [INSURANCE] 可買保險（主注½，2:1）
  → [PLAYER_DECISION] Hit/Stand/Double/Split/Cash Out
     ├ 爆牌(>21) → [BUST] 直接結算
     ├ 21 → [AUTO_STAND]
     ├ Split → [SPLIT_HAND_1] → [SPLIT_HAND_2]
  → [DEALER_REVEAL] 翻暗牌
  → [DEALER_DRAW] 依桌規補牌
  → [SETTLEMENT] 計算+派彩 → [IDLE]
```

### C1.2 牌值計算
| 牌面        | 點數           |
| --------- | ------------ |
| 2–10      | 面值           |
| J / Q / K | 10           |
| A         | 1 或 11（取有利值） |

- **Soft Hand**：含 A 且可算 11 不爆（A+6 = Soft 17）
- **Hard Hand**：不含 A，或 A 只能算 1（A+6+K = Hard 17）

### C1.3 標準決策選項
| 動作          | 條件        | 說明                   |
| ----------- | --------- | -------------------- |
| Hit         | 任何時候      | 要一張                   |
| Stand       | 任何時候      | 停牌                   |
| Double Down | 初始兩張      | 加倍注碼，只補一張            |
| Split       | 兩張同點      | 分兩手各補一張；通常限一次        |
| Surrender   | 初始兩張（部分桌規） | 放棄半注回收               |
| Insurance   | 荷官明牌 A    | 額外注（主注½），荷官 BJ 賠 2:1 |
| Cash Out    | 特定變異      | 提前兌現（動態倍率）           |

### C1.4 派彩表（標準）
| 情況                 | 賠率             |
| ------------------ | -------------- |
| Natural BJ（前兩張=21） | **3:2**        |
| 一般勝                | 1:1            |
| 平局 Push            | 退注             |
| 荷官 BJ（無保險）         | 輸主注            |
| 保險中獎               | 2:1            |
| 加倍後勝               | 1:1（實贏 2× 主注）  |

### C1.5 荷官規則
| 規則                   | 說明                 |
| -------------------- | ------------------ |
| Stand on Hard 17     | 多數桌規標準             |
| Hit on Soft 17 (H17) | 對莊家有利，莊優勢 +約0.2%   |
| Peek（美式）             | 荷官查暗牌，若 BJ 立即結算    |
| No Peek（歐式）          | 荷官不查，玩家須完牌後才知      |

## C2 變異玩法全覽

| 玩法                | 核心差異                                                       |
| ----------------- | ---------------------------------------------------------- |
| Infinite Blackjack | 共享手牌；Four Card Final；Cash Out 0.4–1.77×；Six Card Charlie；CSM 無牌靴。邊注：Perfect Pairs / 21+3 / HOT3 / Bust Bonus |

## C5 決策按鈕 UX

**排列（由左至右）**：`[Split] [Double] [Hit] [Stand]`
- Hit/Stand 最常用 → 最顯眼（右側）；Double/Split 條件滿足才亮；Cash Out/Surrender 特殊條件才出現，不佔常態位置。

**按鈕狀態**：Default / Hover(放大高亮) / Active(按下縮放) / Disabled(灰化降透明) / Selected(保險 Yes/No)。

**禁用條件（Disabled 需 Tooltip 說明原因）**
| 按鈕            | 禁用條件                          |
| ------------- | ----------------------------- |
| Double        | 餘額不足 / 已 Hit 過 / Split Aces 後 |
| Split         | 兩張不同點 / 已 Split 過 / 餘額不足      |
| Cash Out      | Aces Split 後 / 未達條件           |
| Insurance YES | 餘額 < 主注½                      |

## C6 Split 多手牌版型

排列：兩手並列、左右對稱；當前決策手明確高亮邊框；

## C7 直播介面：Zone 分區 + 荷官互動

## C8 牌卡辨識輔助 + Mobile 版型

**牌面點數 Overlay**：疊於對應實體牌正上方，不蓋牌面超過 30%；黑描邊白粗體；發牌後即現，翻牌後更新；Soft Hand 顯示雙值「7│17」。暗牌翻牌前「?」灰色佔位，翻牌 300–400ms 翻轉。

**Mobile 直式**
```
┌─────────────────┐  ← 全寬 375px
│  直播畫面 (16:9) │  ← 上方 55–60%
├─────────────────┤
│ 荷官牌 + 分數    │  ← 固定資訊列 40px
├─────────────────┤
│ 玩家牌 + 分數    │  ← 固定資訊列 50px
├─────────────────┤

├─────────────────┤
│ 籌碼 + 決策鈕     │  ← 固定底部 safe-area
└─────────────────┘
```
**Mobile 橫式**：左側 65% 直播；右側固定面板放荷官牌/玩家牌+分數；底部籌碼 + 決策鈕。
